# Security Policy

## Supported versions

Security fixes are applied to the latest revision of the `main` branch until versioned releases are published.

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability. Use GitHub's private vulnerability reporting for this repository:

1. Open the repository's **Security** tab.
2. Choose **Advisories**.
3. Select **Report a vulnerability**.

Include a clear description, affected files or versions, reproduction steps, impact, and any suggested mitigation. Avoid including unrelated personal data or credentials. You should receive an initial response within seven days.
